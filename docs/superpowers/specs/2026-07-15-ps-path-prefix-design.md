# PanSou `/ps` 统一路径前缀设计

## 目标

部署后，10486 端口对外暴露的所有应用请求统一使用 `/ps` 前缀：页面入口为 `/ps/`，静态资源位于 `/ps/assets/...`，API 位于 `/ps/api/...`，插件 Web 路由位于 `/ps/<plugin>/...`。

根路径 `/` 仅用于重定向到 `/ps/`。原有 `/api/...`、`/assets/...` 和插件根路径不再对外提供。

## 当前结构

`docker-compose.yml` 使用 `ghcr.io/fish2018/pansou-web:latest`。该镜像内部由 Nginx 在 80 端口提供前端文件，并把 `/api` 与插件路径转发给同容器内监听 8888 的 Go 服务。前端构建产物使用 `/assets/...`、`/favicon.ico` 和 `/api/...` 等根绝对路径。

本仓库没有该镜像对应的前端源码，因此不直接修改镜像内的压缩产物或启动脚本。

## 架构

在 Compose 中增加独立的 Nginx `gateway` 服务：

- 宿主机的 `10486` 只映射到 `gateway:80`。
- `pansou` 不再发布宿主机端口，只通过 Compose 默认网络向网关暴露 80 端口。
- 网关将 `/` 返回 302 到 `/ps/`。
- 网关只代理 `/ps/` 下的请求；代理到 `pansou:80` 时剥离 `/ps`。
- 未带 `/ps` 的其他路径返回 404。

应用内部仍使用原有 `/api` 和插件路径，因此 Go 服务、镜像内健康监控及插件注册逻辑无需改变。

## 前端路径改写

上游前端构建产物包含根绝对路径。网关对文本前端响应执行有限的响应改写：

- HTML 中根绝对 `src`、`href` 和 `action` 改为 `/ps/...`。
- JavaScript 中应用自己的 `/api/...` 请求改为 `/ps/api/...`。
- 已知插件 Web 请求的根路径也改为 `/ps/<plugin>/...`。
- CSS 中如存在根绝对资源 URL，则改为 `/ps/...`。

网关向上游清除 `Accept-Encoding`，确保需要改写的文本不是压缩内容；对外仍可由网关启用 gzip。API JSON 不参与改写，避免改变搜索结果或外部链接数据。

改写规则只针对引号、HTML 属性或 CSS `url()` 中的根绝对应用路径，不对 `http://`、`https://`、网盘链接和普通 JSON 内容进行全局替换。

## 请求流

1. 浏览器访问 `http://host:10486/`，网关返回 302，目标为 `/ps/`。
2. 浏览器访问 `/ps/`，网关向上游请求 `/`。
3. 网关把 HTML 中的静态资源地址改写为 `/ps/assets/...` 和 `/ps/favicon.ico`。
4. 浏览器获取 `/ps/assets/...`，网关剥离前缀后向上游请求 `/assets/...`。
5. 前端发起 `/ps/api/...`，网关剥离前缀后向上游请求 `/api/...`。
6. 插件页面及其请求遵循同样规则。

## 错误与兼容行为

- `/`：302 跳转到 `/ps/`。
- `/ps`：301 或 302 规范化到 `/ps/`。
- `/ps/...`：由上游应用处理；上游状态码原样返回。
- `/api/...`、`/assets/...`、旧插件路径及其他未匹配路径：404。
- 网关无法连接应用时：返回标准 502，不回退到旧路径。

## 部署与健康检查

`pansou` 保留镜像自带的内部健康检查。`gateway` 增加针对 `/ps/api/health` 的健康检查，并依赖 `pansou` 启动。`autoheal` 继续监控标记为 `autoheal=true` 的服务。

## 验证标准

- `docker compose config` 成功。
- 网关 Nginx 配置检查成功。
- `GET /` 跳转到 `/ps/`。
- `GET /ps/` 返回 HTML，HTML 引用均带 `/ps`。
- HTML 引用的 JS、CSS、favicon 可成功获取。
- `GET /ps/api/health` 返回 200。
- `GET /ps/api/search` 能到达后端并返回后端响应。
- 一个已启用插件的 `/ps/<plugin>/...` 请求能够到达插件路由。
- `GET /api/health`、`GET /assets/...` 和旧插件路径返回 404。
- 容器重建后以上行为保持不变。

## 范围限制

- 不修改外部第三方请求或搜索结果中的 URL。
- 不新增第二套兼容 API。
- 不引入或维护 `pansou-web` 前端源码。
- 不修改 Go 服务的内部路由前缀；`/ps` 是部署边界的统一外部前缀。
